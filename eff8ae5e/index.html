Maven的安装与配置

​ Maven是一个项目管理和整合的工具。Maven为开发者提供了一套完整的构建生命周期框架。开发团队基本不用花多少时间就能自动完成工程的基础构建配置，因为Maven使用了一个标准的目录结构和一个默认的构建生命周期。在创建报告、检查、构建和测试自动配置时，Maven可以让开发者的工作变得更简单。

一、安装本地Maven

  1. 点此进入maven官网下载

  2. 选择左侧Download

  1. 下载apache-maven文件，选择自己需要的版本

  2. 下载完成后进行解压

  3. 然后配置环境变量

    • 增加系统变量:MAVEN_HOME = F:\MAVEN\apache-maven-3.6.0
    • 增加系统变量:path = %MAVEN_HOME%\bin

  4. 然后win+R 运行cmd 输入 mvn -version，如图所示则配置成功

二、配置settings文件

  1. 在F:\MAVEN\apache-maven-3.6.0\conf下可以找到settings文件，打开(嫌麻烦的直接看最后一步)

  2. 找到第52行，这里是maven默认的仓库

  3. 复制 /path/to/local/repo后放到注释外将中间改成你需要仓库存放的位置

  4. 因为国外的服务器下载jar包很慢所以我们改为阿里云服务器（大约在150行左右），这两个仓库只用选一个

    <!-- 阿里云仓库 -->
    <mirror>
        <id>alimaven</id>
        <mirrorOf>central</mirrorOf>
        <name>aliyun maven</name>
        <url>http://maven.aliyun.com/nexus/content/repositories/central/</url>
    </mirror>
 或者
  <mirror>
    <id>nexus-aliyun</id>
    <mirrorOf>*</mirrorOf>
    <name>Nexus aliyun</name>
    <url>http://maven.aliyun.com/nexus/content/groups/public</url>
 </mirror>

  5. 如图，要夹在两个mirrors标签之间

  6. 在最后配置jdk，也要夹在两个profiles标签之间

    <!—java版本 –>

    jdk-1.8 true 1.8 
      <properties>
    <maven.compiler.source>1.8</maven.compiler.source>
    <maven.compiler.target>1.8</maven.compiler.target>
    <maven.compiler.compilerVersion>1.8</maven.compiler.compilerVersion>
  </properties>

  7. 配置完成，在命令行输入mvn help:system测试，看到下载链接里面是ailiyun的链接表示配置成功
